Overview

CodeDost API Documentation

Complete reference for the CodeDost backend REST API. Handles user authentication, usage tracking across three tiers, and Pro subscription management via Gumroad.

The AI analysis itself runs directly from the frontend to Groq/Gemini/OpenRouter. This backend handles authentication, enforcing usage limits, and logging — it does NOT proxy AI requests.

Base URL

Production
https://codedost-backend.onrender.com
Local Development
http://localhost:5000

All endpoints are prefixed with /api. Example: POST /api/auth/login

Authentication

CodeDost uses JWT (JSON Web Tokens) with a two-token system for security.

Access Token

Short-lived (15 minutes). Send in every protected request.

Option A — Authorization Header (recommended for API clients)
Authorization: Bearer <accessToken>
Option B — Cookie (automatic for browser apps)
// Set automatically on login. Sent automatically by browser. // httpOnly = JS cannot read it. secure = HTTPS only. Cookie: accessToken=<token>
Refresh Token

Long-lived (7 days). Stored in an httpOnly cookie. Used only to get a new access token when it expires. Never send this manually.

When the frontend receives a 401 TOKEN_EXPIRED response, call POST /api/auth/refresh to get a new access token silently, then retry the original request.

Error Handling

All errors follow the same JSON format. Always check success: false and read error for the message.

Error Response Format
{ "success": false, "error": "Human-readable error message", "code": "MACHINE_READABLE_CODE", // optional "upgradeUrl": "https://...", // only on limit errors "loginPrompt": true // only for guest limits }
HTTP Status Codes
200Success
201Created
400Bad request / validation
401Not authenticated
403Forbidden / line limit
409Conflict (email exists)
429Monthly limit reached
500Server error
Machine-Readable Codes
Code Meaning Action
TOKEN_EXPIRED Access token expired Call /auth/refresh then retry
LINE_LIMIT_EXCEEDED Code too long for tier Show upgrade prompt
MONTHLY_LIMIT_REACHED Free user hit 20/month Show upgrade prompt
GUEST_LIMIT_REACHED Guest hit 5/month Show login/register prompt
PRO_REQUIRED Feature needs Pro Show Pro upgrade link
Authentication

POST /api/auth/register

Create a new user account. Returns access token + sets httpOnly cookies.

POST /api/auth/register No auth required ▾
Request Body
Field Type Required Description
name String required Full name. 2–50 characters.
email String required Valid email address. Must be unique.
password String required Minimum 8 characters. Hashed before storage.
university String optional University name for profile.
Request Example
fetch('/api/auth/register', { method: 'POST', headers: { 'Content-Type': 'application/json' }, credentials: 'include', // Required for cookies body: JSON.stringify({ name: 'Ahmed Khan', email: 'ahmed@example.com', password: 'mypassword123', university: 'FAST NUCES Lahore' }) });
Success Response — 201 Created
{ "success": true, "accessToken": "eyJhbGciOiJIUzI1NiIs...", "user": { "id": "64f3a2b1c9e7f8d0a1b2c3d4", "name": "Ahmed Khan", "email": "ahmed@example.com", "tier": "free", "usage": { "used": 0, "limit": 20, "remaining": 20, "maxLines": 1000 }, "preferredMode": "urdu", "preferredLang": "python" } }
Cookies accessToken and refreshToken are set automatically as httpOnly cookies. The browser will send them automatically on every subsequent request.

POST /api/auth/logout

POST /api/auth/logout JWT Required ▾

Logs out the current device. Removes the refresh token from the database and clears both cookies. Other devices remain logged in.

Request Example
fetch('/api/auth/logout', { method: 'POST', credentials: 'include' // No body needed — token comes from cookie });
Success Response — 200
{ "success": true, "message": "Logged out successfully." }

POST /api/auth/refresh

POST /api/auth/refresh Refresh Cookie ▾

Gets a new access token silently. Reads the refresh token from the refreshToken cookie. Rotates the refresh token (old one invalidated, new one issued).

Call this automatically when any request returns 401 TOKEN_EXPIRED, then retry the original request with the new token.
Auto-Refresh Pattern (add this to your frontend)
async function apiFetch(url, options = {}) { options.credentials = 'include'; let res = await fetch(url, options); // If token expired, refresh silently and retry once if (res.status === 401) { const data = await res.json(); if (data.code === 'TOKEN_EXPIRED') { const refreshRes = await fetch('/api/auth/refresh', { method: 'POST', credentials: 'include' }); if (refreshRes.ok) { const { accessToken } = await refreshRes.json(); options.headers = { ...options.headers, Authorization: `Bearer ${accessToken}` }; res = await fetch(url, options); // Retry } } } return res; }
Success Response — 200
{ "success": true, "accessToken": "eyJhbGciOiJIUzI1NiIs..." }

Analysis

POST /api/analyze

Called before making an AI API call. Checks the user's quota, logs the analysis, and increments usage counters. Works for guests, free, and pro users.

POST /api/analyze Optional JWT ▾
Call this endpoint BEFORE making the Groq/Gemini request. If it returns success, proceed with the AI call. If it returns 429 or 403, show the appropriate message.
Request Body
Field Type Required Description
code String required The code being analyzed. Line count is checked against tier limit.
language String optional python | javascript | java | cpp | html | sql
errorMessage String optional The error message string. Stored for analytics.
mode String optional urdu | mixed | english
aiProvider String optional groq | gemini | openrouter
Integration Pattern — use this in analyzeCode()
async function analyzeCode() { const code = document.getElementById('code-input').value.trim(); // Step 1: Check quota with backend const quotaRes = await fetch(`${API_URL}/api/analyze`, { method: 'POST', headers: { 'Content-Type': 'application/json' }, credentials: 'include', body: JSON.stringify({ code, language: currentLang, mode: currentMode }) }); const quotaData = await quotaRes.json(); if (!quotaData.success) { if (quotaData.code === 'GUEST_LIMIT_REACHED') { showLoginModal(); // Prompt to register } else if (quotaData.code === 'MONTHLY_LIMIT_REACHED') { showUpgradeModal(); // Prompt to upgrade to Pro } else { showToast('error', quotaData.error); } return; } // Step 2: Proceed with AI call (Groq/Gemini/OpenRouter) // ... existing AI fetch code ... }
Success Response — 200
{ "success": true, "message": "Analysis logged. Proceed with AI call on frontend.", "quota": { "tier": "free", "remaining": 12, // null for Pro (unlimited) "maxLines": 1000 } }
Error — Monthly limit reached (429)
{ "success": false, "error": "Monthly limit reached. Free accounts get 20 analyses per month.", "code": "MONTHLY_LIMIT_REACHED", "limit": 20, "used": 20, "resetsOn": "2025-04-01T00:00:00.000Z", "upgradeUrl": "https://sherazramzan.gumroad.com/l/cfevjo" }
Error — Code too long (403)
{ "success": false, "error": "Code too long. Guest users support up to 300 lines.", "code": "LINE_LIMIT_EXCEEDED", "limit": 300, "actual": 450, "loginPrompt": true }

GET /api/analyze/quota

GET /api/analyze/quota Optional JWT ▾

Check remaining quota without consuming it. Call on app load to show the usage counter in the UI.

{ "success": true, "tier": "free", "remaining": 13, "used": 7, "limit": 20, "maxLines": 1000, "resetsOn": "2025-04-01T00:00:00.000Z" }

{ "success": true, "tier": "pro", "remaining": null, // null means unlimited "maxLines": null // null means unlimited }

{ "success": true, "tier": "guest", "remaining": 3, "used": 2, "limit": 5, "maxLines": 300, "loginPrompt": true // show login nudge when remaining <= 2 }

Activates Pro tier by verifying a Gumroad license key. After purchasing at gumroad.com/l/cfevjo, the user receives a license key by email and enters it here.

POST /api/subscription/activate JWT Required ▾
Request Body
Field Type Required Description
licenseKey String required Gumroad license key from the purchase confirmation email.
Request Example
fetch('/api/subscription/activate', { method: 'POST', headers: { 'Content-Type': 'application/json' }, credentials: 'include', body: JSON.stringify({ licenseKey: 'XXXX-XXXX-XXXX-XXXX' }) });
Success Response — 200
{ "success": true, "message": "Pro subscription activated! Enjoy unlimited access.", "user": { "tier": "pro", "subscription": { "isActive": true, "startDate": "2025-03-20T10:00:00.000Z", "endDate": null, "plan": "lifetime" } } }
